Questions

  1. Assume you are comparing three different algorithms to solve some problem and you have determined the worst-case time equations for each:

    • Algorithm 1: T(N) = N2 + N + 20
    • Algorithm 2: T(N) = 1000N + 496
    • Algorithm 3: T(N) = 20Nlog2N + 3N + 5

    Answer the following questions regarding these three algorithms.

    1. What is the worst-case time complexity (Big-O) for each algorithm?
    2. Which algorithm has the lowest worst-case time complexity?
    3. Should the algorithm with the lowest order of complexity always be used? Briefly explain why or why not.

  2. Suppose the DblListnode<E> class is implemented as in the readings. The following class implements the ListADT using a circular doubly-linked chain of nodes:

    
public class CircularDblLinkedList<E> implements ListADT<E> {

    // Reference to the first node (position 0) of a circular doubly-linked
    // chain of nodes. Recall that the next reference of the last node
    // references the first node and the prev reference of the first node
    // references the last node making the chain of nodes circular.
    private DblListnode<E> items;
    // The number of items in the list
    private int numItems;

    public CircularDblLinkedList() {
        // Initially the list is empty
        items = null;
        numItems = 0;
    }

    // Return the item at position pos if pos is valid, otherwise throw
    // IndexOutOfBoundsException. This method traverses the fewest nodes
    // possible while locating the node at pos.
    public E get(int pos) {
        // IMPLEMENT THIS
    }
}

    Complete the get() method according to its specification. For full credit, your solution

    • must locate the node at pos in the most efficient manner possible (i.e., call getNext() and getPrev() as few times as possible);
    • must not modify the DblListnode class, the CircularDblLinkedList class, or any method or field declarations therein, except the body of the get() method;
    • must not change the contents or ordering of the nodes in the list during execution.


  3. Briefly answer the following questions assuming the list has N nodes. Full credit is given for concise answers.

    1. What is the exact worst-case number of getNext() and getPrev() calls required to locate a node in your implementation of get() from Question 2?
    2. What is the worst-case time (Big-O) of your implementation?
    3. Suppose we had used a singly linked list instead of a doubly linked list. What would be the exact worst-case number of getNext() calls required to locate a node in this case?
    4. What would be the worst-case time (Big-O) in the singly linked case?
    5. When is each implementation better than the other? Is one choice always best?
